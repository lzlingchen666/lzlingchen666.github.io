数据结构————链表

数据结构

数据结构有什么用呢?有时候我们学的时候就是这么想的对吧,也对有时候我们就是这么的难以想象数据结构这种有麻烦又费脑子的东西有什么用,用数组不是就够了吗?还要什么数据结构啊!

这么像我们就错了,数据结构绝对是计算机发展史上最伟大的突破之一,它的伟大就在于突破了人们对于计算机数据的表示形式发生了变化,这是一种思想上的突破,这代表我们不在局限于那种简单,单一的数据形式,我们更多的是结合我们的思维对数据的存储方式进行设计,由此诞生了链表,栈,队列,树,集合,图这些经典的,基础的数据结构,几乎所有的数据结构都是在这几种基础结构上演变而来的,所谓万变不离其宗,我们学习数据结构更多的是要学习里边的思想,而灵活地应用,而并不是死记硬背这几种数据结构,死记硬背是没有用的,因为数据结构这东西是很灵活的,如果不能理解那到头来只要稍微的有些许的变动,就无能为力了.

还有一点很重要,那就是数据结构是与算法挂钩的,我们学的时候往往学的是《数据结构与算法分析》,这也充分表明数据结构与算法分析之间的联系相当的紧密,一个好的算法往往是需要数据结构来搭配的,而数据结构的生成过程也离不开算法,本质上数据结构也能算是一种另类的算法.

所以只这个系列中我会将我所学的数据结构进行一个整合来分享,同时也是为了提高我自己的水平更好的掌控这门学科.

链表

何为链表?

所谓链表,顾名思义就是像一条链子一样的的,我更愿意将它比喻为由一段一段锁链和结点所组成的一种数据结构.它与数组不同,数组就像是一堆连续排列的结点,它在地址空间中是连续的就像是我们有的地方的别墅区,一排排的别墅连在一起一样,我们可以很轻易的就确定某一栋别墅的位置,因为他们都是连在一起的.

但是链表就不同了.打个比方,有一个秘密的组织,组织成员只能知道他的下一个人员的所有信息2和居住的位置,其他的一概不知,当有许多的成员的时候,这个组织的形式就类似于单链表,这个组织中的任何一个成员都只知道他的下属的居住位置和所有信息,这样一个一个接下去,这种模式就是单链表.单链表在地址空间中是不连续的,它由结点中的指针来确定下一个元素的位置,就像是锁链一样,一个接一个,二者之间可以很遥远,也可以里的很近,只是锁链的长度问题,所以相比属猪数组而言,这种形式更加的自由,但是也有一个问题,我们要访问某一个元素,我们就得找到第一个元素顺藤摸瓜一个一个的往下找,因为他们在地址空间中是不连续的.

链表与数组的优劣势

  1. 查找:从查找上来看,链表是没有数组方便的,因为数组在空间中的连续性就决定了他查找起来很方便,我们只需要知道头结点的位置和每一个节点所占空间的大小,就能够轻易地找到任何一个元素的所在之地.反观链表,他要查找某个结点,就必须要从第一个结点开始一个一个的往下找,这样要做的工作量就大了,尤其是有海量的数据时.
  2. 删除/添加:在这一点,数组是比不了链表的,数组每天夹或者删除一个结点都需要将后面的结点全部向前或者向后移动,这样的效率是很低的,尤其是数据比较多的时候,二连表呢,他只需要找到那个元素,如果是添加,就找到该位置前面一个结点,将前面的结点的指针指向的结点赋值给这个元素的指针,然后将前面那个结点的指针指向需要添加的节点(顺序很重要,绝对不能颠倒!)就是这么简单.删除的话也是类似.
  3. 数组在空间中是连续的,所以假如说内存不足(这种情况很少见)或者说需要的内存太大,就会出现空间分配失败的情况,而链表则没有这种情况.
  4. 遍历:两者没有什么区别

以上就是一张单链表的C语言实现,当然我没有编写main()方法,因为这都是比较简单的,淡然链表的强大之处远远不止于此,由于指针的使用,使得链表很灵活,可以说你能想到就能够实现,而且这里还没有使用什么高端的技术.后面我还会向大家介绍各种各样的链表,例如:双向链表,循环链表,循环双向链表,还有哈希表……这些方法在Java里面都是已经被封装好了,可以直接查看源代码来研究Java是如何实现这些数据结构的.
